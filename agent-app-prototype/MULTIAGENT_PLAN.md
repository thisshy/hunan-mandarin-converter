# 湘语通 App 原型的多 Agent 任务设计

本原型对应的是你当前 `hunan-mandarin-converter` 项目的重构方向，不修改旧项目文件，只在新目录中输出新的 App 设计。

## Agent 拆分

1. 产品梳理 Agent
   - 读取现有 `index.html`、`app.js`、`server.js`
   - 提炼核心能力：双向转译、候选表达、词库审核、评测记录
   - 输出 App 的页面结构和优先级

2. 交互架构 Agent
   - 把网页的三个大区块重组为四个移动端标签页
   - 标签页分别为：`转译`、`词库`、`审核`、`洞察`
   - 保留“场景 + 方言 + 候选结果 + 评分闭环”的主链路

3. 视觉设计 Agent
   - 负责色彩、版式、信息密度和移动端设备呈现
   - 视觉方向：米白纸感底色 + 深青主色 + 橙色强调
   - 目标是从原网页的“功能台”升级成更像产品的“App 屏幕”

4. 原型实现 Agent
   - 在独立目录 `agent-app-prototype` 中实现静态高保真原型
   - 使用静态数据模拟后端，不依赖原项目运行
   - 保证可以直接打开预览

5. 预览验证 Agent
   - 打开新原型页面
   - 检查不同标签页是否可切换
   - 截图输出当前设计效果

## 数据与交互策略

- 静态化的数据
  - 场景说明
  - 方言示例结果
  - 词库建议
  - 审核条目
  - 评测记录

- 保留为模拟态的交互
  - 切换方言 / 场景
  - 生成不同候选表达
  - 点击候选结果替换主结果
  - 底部标签切换不同工作台

- 暂不接入真实后端的能力
  - 真正的词库导入导出
  - 管理员 token 校验
  - 数据持久化
  - 真实评分写入与后台审批

## 这版 App 的定位

这不是简单把网页缩小成手机屏，而是把现有能力包装成一个更适合演示、申报和后续产品化的 App 概念稿。
